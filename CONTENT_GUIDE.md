## 📁 文件在哪里

所有你需要编辑的内容都在 `content/` 文件夹下：

```
content/
├── blog/          ← 博客文章放这里
├── projects/      ← 项目介绍放这里
└── pages/         ← 「关于我」等固定页面放这里
```

你只需要用 VS Code 打开对应的文件夹，创建或修改 `.md` 文件即可。

## 添加博客文章

### 第一步：创建文件

在 `content/blog/` 下新建一个文件，文件名格式：

```
YYYY-MM-文章英文简称.md
```

例如：`2026-08-summer-tour-report.md`

### 第二步：写内容

文件开头需要一个「前言」（frontmatter），用三个 `---` 包裹。然后是正文（写法和公众号编辑器差不多）。

```markdown
---
title: "你的文章标题"
date: "2026-08-15"
description: "这篇文章的一句话摘要，会出现在卡片和搜索引擎结果中。"
tags: ["文化遗产", "导览", "随笔"]
coverImage: "/images/blog/封面图.jpg"
draft: false
featured: false
---

## 第一个小标题

这里是正文内容。Markdown 的写法非常简单：

- 用 `##` 开头就是小标题
- 用 `-` 开头就是列表
- **粗体** 用两个星号包裹
- [链接文字](https://...) 就是超链接

插入图片：
![图片说明](/images/blog/图片名.jpg)

> 这是引用块，用于引用别人的话或强调。

---

### 一些技巧

用 `---` 分隔线来分隔不同部分。

结束。
```

### 第三步：填好前言信息

| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 文章标题 | `"博物馆导览小程序的开发心得"` |
| `date` | 发布日期 | `"2026-08-15"` |
| `description` | 摘要（1-2 句话） | `"分享我们为某博物馆做的导览小程序……"` |
| `tags` | 标签 | `["小程序", "博物馆"]` |
| `coverImage` | 封面图路径 | `"/images/blog/cover.jpg"`（可选，没有就留空） |
| `draft` | 是否草稿 | `false` = 发布；`true` = 不显示 |
| `featured` | 是否在首页展示 | `false` = 只在博客列表；`true` = 同时出现在首页 |

## 添加项目

### 和博客几乎一样，在 `content/projects/` 下创建文件：

```markdown
---
title: "项目名称"
date: "2026-08-01"
description: "项目的一句话描述"
tags: ["小程序", "AR"]
category: "mini-program"
coverImage: "/images/projects/demo.jpg"
images:
  - "/images/projects/demo-1.jpg"
  - "/images/projects/demo-2.jpg"
demoUrl: "https://..."
sourceUrl: ""
draft: false
featured: true
---

## 项目背景

描述项目背景……

## 主要功能

### 功能一

描述……
```

### 项目特有的字段

| 字段 | 说明 | 可选值 |
|------|------|--------|
| `category` | 项目分类 | `"mini-program"` 小程序 / `"web"` 网站 / `"ai"` AI / `"design"` 设计 / `"other"` 其他 |
| `images` | 详情页的截图集 | 列表格式，每个一行 |
| `demoUrl` | 线上体验地址 | 可选 |
| `sourceUrl` | 源码地址 | 可选 |

## 修改关于页面

编辑 `content/pages/about.md` 文件。

这个文件没有多余的前言字段，直接写 Markdown 正文即可：

```markdown
---
title: "关于澄川鸮舍"
description: "一个专注文化遗产数字化的小型工作室"
---

## 自我介绍

这里是你的自我介绍……

## 工作经历

- 2021 - 至今：……
- 2019 - 2021：……
```

## 修改网站信息

如果你想改网站名字、slogan、导航菜单等全局信息，编辑 `src/lib/constants.ts`：

```typescript
export const SITE = {
  name: "澄川鸮舍",          // ← 改名字在这里
  tagline: "一川琥珀 万物有灵", // ← 改 slogan 在这里
  // ...
};
```

## 图片管理

所有图片放在 `public/images/` 下：

```
public/images/
├── logo.png              ← 网站 logo
├── og-default.png        ← 分享链接时的默认图
├── blog/                 ← 博客文章的配图
└── projects/             ← 项目截图
```

在 Markdown 中引用图片时，路径以 `/images/` 开头：
```
![说明文字](/images/blog/我的图.jpg)
```

## 如何上线

当你完成了内容编辑，需要做这几步来让网站更新：

```bash
# 1. 保存所有文件

# 2. 提交到 Git
git add .
git commit -m "新增文章：xxx"

# 3. 推送到 GitHub
git push

# Vercel 会自动检测到推送，自动部署（约 30 秒完成）
```

如果你还没有设置 Git 和 Vercel，参考 `README.md` 中的部署步骤。

## 💡 小贴士

1. **文件名用英文**——中文文件名在不同系统上可能出问题
2. **先设为草稿**——如果文章还没写完，把 `draft: true`，它就不会出现在网站上
3. **封面图推荐尺寸**——博客封面 1200×675（16:9），项目封面 800×600（4:3）
4. **修改后本地预览**——在终端运行 `npm run dev`，浏览器打开 `http://localhost:3000` 就能看到效果
5. **遇到问题？**——直接在对话里问我，比如「怎么在文章里插入视频？」或者「我想加一个新页面」

## 📋 Markdown 速查表

| 你想要的 | 写法 |
|----------|------|
| **粗体** | `**粗体**` |
| *斜体* | `*斜体*` |
| 小标题 | `## 标题`（## 越多，标题越小） |
| 列表 | `- 项目一` |
| 数字列表 | `1. 第一` |
| 链接 | `[文字](网址)` |
| 图片 | `![描述](图片路径)` |
| 引用 | `> 引用文字` |
| 分隔线 | `---` |
| 行内代码 | `` `代码` `` |
